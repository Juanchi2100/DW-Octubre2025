Día 27 - Funciones

  • Nivel 1 - 3

1. click

Target: elementos interactivos (botón, enlace, etc.)

Se dispara cuando el usuario presiona y suelta el botón principal del ratón sobre un elemento. Es el evento más usado para acciones de usuario (ej. ejecutar una función al pulsar un botón).

2. dblclick

Target: elementos

Se activa cuando el usuario hace doble clic. Útil para acciones que deben requerir doble confirmación. No recomendado para interfaces táctiles.

3. mouseover

Target: elemento que recibe el puntero

Cuando el cursor entra en el área de un elemento. A diferencia de mouseenter, mouseover burbujea (se propaga a ancestros).

4. mouseout

Target: elemento que pierde el puntero

Se dispara cuando el cursor abandona un elemento. Útil para ocultar tooltips o revertir estilos aplicados en mouseover.

5. mousemove

Target: elemento

Se ejecuta continuamente mientras el ratón se mueve dentro del elemento. Úsalo con precaución por rendimiento; para rastrear la posición del cursor o animaciones finas.

6. mousedown / mouseup

Target: elemento

mousedown ocurre cuando el botón del ratón se presiona; mouseup cuando se suelta. Útiles para detectar arrastres o acciones que dependen del tiempo de pulsación.

7. contextmenu

Target: elemento

Se dispara al intentar abrir el menú contextual (clic derecho). Puedes prevenirlo con event.preventDefault() para crear menús personalizados.

8. keydown

Target: elemento con foco (documento o input)

Ocurre cuando se presiona una tecla. Devuelve información como event.key. Ideal para atajos de teclado o validar entradas. Se dispara repetidamente si la tecla se mantiene pulsada.

9. keyup

Target: elemento con foco

Se dispara al soltar la tecla. Útil para acciones tras completar la pulsación (ej. validar al terminar de escribir).

10. keypress

Target: elemento con foco

Evento histórico para teclas imprimibles; está deprecado en favor de keydown/keyup. Evita usarlo en código nuevo.

11. input

Target: campos de formulario

Se ejecuta cuando cambia el valor de un input, textarea o elemento editable. Ideal para validación en tiempo real y autocompletar.

12. change

Target: campos de formulario

Se dispara cuando el valor del campo cambia y el control pierde foco (o tras selección, en selects). Útil para acciones que deben ocurrir tras confirmar la edición.

13. submit

Target: <form>

Ocurre cuando se envía un formulario. Puedes usar event.preventDefault() para interceptar el envío y procesar datos vía JS (fetch/AJAX).

14. focus

Target: elementos enfocables (inputs, botones)

Se disparar cuando un elemento recibe foco. No burbujea; para capturarlo en ancestros, usa focusin.

15. blur

Target: elementos enfocables

El evento inverso de focus: ocurre al perder el foco. Útil para validación al salir del campo.
